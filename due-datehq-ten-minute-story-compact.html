10 minutes Demo Day compact

开场

把 CPA 从截止日期里解放出来。

DueDateHQ 不是更漂亮的日历。它把客户画像、官方规则、截止日、沟通和下一步动作连成一个可运行的工作闭环。

今天只证明一件事

上传客户表之后，系统能生成工作台；规则变化之后，系统能指出受影响客户；CPA 最后只做判断。

Deadline intelligence CPA workflow AI-assisted Live backend demo

目标客户与痛点

Sarah 不是缺工具。她缺的是一个每天告诉她先做什么的系统。

60个客户，一个 CPA owner 加兼职助理管理全部事项。
23个跨州客户，州、税种、实体类型组合持续膨胀。
400+联邦和州层面的截止日、延期和周期性事项。
12h每周浪费在追踪、更新、确认和重新排序上。

这个市场不是没有 due date 工具，而是现有工具大多停留在“存日期”。真正昂贵的是每天反复判断：这个客户现在到底该推进、等待、审核，还是归档。

市场空白

File In Time 像数据库，Karbon 像工作流。Sarah 需要的是规则驱动的工作台。

传统 due date tracker

擅长存表、频率和 rollover，但规则变化、客户状态和客户沟通仍然靠人维护。

通用 practice management

擅长团队协作和 checklist，但不理解 50 州税务规则、延期、公告和客户适用性。

DueDateHQ

以 deadline 为核心对象，把规则变化、客户画像和下一步动作放在同一个分诊面里。

AI 差异化

我们的 AI 不是 chatbot layer。它负责把维护成本从 CPA 身上拿走。

Parse

把 messy CSV 和客户资料解析成结构化 client profile。

Watch

读取 CA / TX / NY 等官方来源，提取规则变化。

Match

把规则变化匹配到真实客户组合和 work item。

Draft

根据 work context 生成客户跟进和下一步动作。

AI 不替代 CPA 的最终判断。它减少规则维护、资料整理、影响分析和文字起草的时间，让 CPA 把注意力留给判断。

Aha Moment

一个客户表格，变成事务所的工作台。

导入客户从竞品或 Excel 上传客户表。
生成画像客户、州、实体类型、税种和联系人结构化。
派生工作系统生成 deadline、task、blocker 和 reminder。
每周分诊Work now / Blocked / Needs review / Overdue。
政策响应官方变化进入 Review，CPA 决定 apply 或 dismiss。

Live Demo 2 · Weekly triage

周一早上，Sarah 打开 Work，不再先做 25 分钟分诊。

Work now现在可以推进的事项。
Blocked缺文件、缺确认、缺 profile 信息。
Needs review政策变化或 notice 需要 CPA 判断。
Overdue已经过期但还没有归档。

打开一个 work item，可以看到 source、reminder、blocker、extension 和 client follow-up。需要客户资料时，AI 只在用户点击 draft 时生成邮件草稿。

Live Demo 3 · Official rule response

州税变化不是新闻摘要，而是受影响客户清单。

当前实现进度

现在不是只有前端。核心 demo 闭环已经有可访问的 live backend。

客户导入与档案

已实现：CSV 上传、后端 preview / apply、字段映射、行级审核、新增 / 更新客户、客户详情和编辑入口。

正在实现：更强的不规范文档解析、duplicate merge 策略和更完整的导入审计记录。

Deadline 与 Work board

已实现：从 client profile 派生 work item，支持 Work now、Blocked、Needs review、Overdue、Archive、extension 和 dashboard payload API。

正在实现：把更多状态修改写入稳定 API，并补齐更完整的历史记录。

官方规则与影响分析

已实现：CA / TX / NY source-specific sync、official source、diff、affected clients 和前端 Review 操作。

正在实现：扩大州税来源覆盖，接入正式调度、异常告警和规则版本管理。

提醒与客户跟进

已实现：reminder timeline、client follow-up、AI draft、notification preview / send-pending API，以及 Settings 里的 live queue。

正在实现：Email / Slack OAuth、可配置 cadence 和真实发送日志。

延期、归档与导出

已实现：File / revoke extension、extended due date、archive、CSV export 入口和报告数据结构。

正在实现：PDF 报告、view-level export package 和更完整的审计证据链。

Ask、API 与 CLI

已实现：Ask backend streaming；API 覆盖 import、dashboard、source sync、notifications；CLI 覆盖 import、client、deadline、task、notice、source、notify、export。

正在实现：把前端所有 write action 收敛成 API / CLI 双入口，方便部署和自动化测试。

Go-to-Market

GTM 不是先投广告，而是先手动把 Sarah 服务到离不开。

目标人群40-150 客户独立 CPA / 小型事务所 owner，跨州客户比例高。
切入方式Concierge onboarding用客户表导入开局，第一周帮她清出 Work queue。
验证标准30 天不想回 Excel她主动问“下周哪些客户也可以导进来”。

早期卖的不是“AI 税务软件”，而是一个具体结果：把客户 deadline 整理好，每天告诉 Sarah 最该处理哪几件。定价可以从 onboarding fee + monthly workspace 开始，再按客户数或 source coverage 扩展。

未来期待

同一个 CPA 用户基础，可以转出三个飞轮。

1

规则库飞轮

用户越多，官方规则变化被更多真实客户组合验证，规则库越准。

2

意图飞轮

CPA 的高频表达会收敛成 Intent Library，Ask 越用越快、越便宜。

3

数据飞轮

客户画像、deadline、状态和审计记录越完整，产品越难被替代。

今天 demo 的是第一个 wedge：deadline intelligence。长期目标是成为 deadline-heavy CPA practice 的 operating layer。

收尾

DueDateHQ 保护的，是 CPA 的判断力。

一个 CPA 最有价值的东西，从来不是她记住了多少截止日期，而是她对客户情况的理解和判断。

规则库更新，不需要她盯着 IRS 官网。优先级排序，不需要她每天早上想 25 分钟。操作记录，不需要她手动更新 Excel。

这些事情系统做了，Sarah 的时间和注意力就留给真正需要她的事：判断、沟通、建议。
